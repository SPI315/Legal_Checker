# Spec: Serving / Config

## Запуск

Основной способ запуска backend:

- `uvicorn app.main:app --reload`

С точки зрения пользователя система должна иметь единый endpoint обработки документа, например `POST /api/documents/process`.

## Конфигурация

Конфигурация передается через `.env` с префиксом `LEGAL_`.

Она должна покрывать:

- параметры приложения;
- настройки LLM provider и модели;
- настройки OCR;
- параметры retrieval;
- параметры observability.

## Ключевые параметры

- `LEGAL_APP_NAME`, `LEGAL_APP_VERSION`
- `LEGAL_LLM_PROVIDER`, `LEGAL_LLM_MODEL`
- `LEGAL_LLM_TIMEOUT_SEC`, `LEGAL_LLM_MAX_RETRIES`
- `LEGAL_RETRIEVER_TIMEOUT_SEC`, `LEGAL_RETRIEVER_TOP_K`
- `LEGAL_ALLOWED_SOURCE_DOMAINS`
- `LEGAL_OCR_ENABLED`

## Секреты

Все API keys и другие секреты передаются только через env.

Дополнительные правила:

- значения секретов нельзя логировать;
- секреты не хранятся в репозитории;
- секреты не должны попадать в статические конфигурационные файлы.

## Версии моделей

Активная модель фиксируется в конфиге и логируется в trace.

Если меняется версия модели:

- это должно отражаться в eval history;
- старые и новые результаты должны быть сравнимы на одном и том же наборе проверок.

Для PoC допускается замена провайдера, если сохраняется единый контракт ответа.
