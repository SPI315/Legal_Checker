# Legal Doc Checker

PoC-система для юриста: автоматическая первичная проверка юридических документов на риски, с анонимизацией персональных данных и разметкой проблемных фрагментов.

## Задача, аудитория и текущая боль

- Для кого: юрист (in-house или консалтинг).
- Задача: ускорить и стандартизировать первичную юридическую проверку документов.
- Боль сейчас:
  - ручная проверка занимает много времени;
  - типовые риски пропускаются из-за объема и дедлайнов.

## Что делает PoC на демо

- Загружает `docx`.
- Извлекает структуру документа (абзацы, offsets, paragraph_id).
- Запускает анонимизацию PII перед внешним LLM/API-вызовом.
- Выполняет анализ рисков (правила + подключаемый LLM).
- Делает сверку:
  - с открытой нормативной базой через web-поиск/подключаемые API;
  - с локальными проектными документами через RAG-логику.
- Показывает в UI:
  - список обнаруженных рисков (тип, комментарий, предложение правки);
  - подсветки в тексте;
  - переход к абзацу по клику.
- Экспортирует результат:
  - JSON-отчет;
  - `docx` с комментариями (MVP-уровень).

## Что PoC НЕ делает (out-of-scope)

- Не дает финального юридического заключения без участия человека.
- Не реализует полноценный enterprise IAM/SSO (в PoC без аутентификации).
- Не гарантирует полноту/актуальность всех норм права во всех юрисдикциях.
- Не является production-ready DLP/SIEM-решением.

## Ключевые ограничения PoC

- Целевая задержка: p95 до 60 секунд на документ среднего размера (15-20 страниц).
- Оригинальный текст документа не хранится в логах/БД; в хранилище только анонимизированные данные.
- Текущая версия backend работает без OCR: поддерживаются `docx` и текстовые `pdf`.

## Backend (MVP)

Добавлен минимальный backend на FastAPI с endpoint для анонимизации.

### Быстрый старт

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

### Конфигурация NER

По умолчанию сервис работает в `regex-only` режиме.
Чтобы включить NER через transformers, укажите имя модели:

```bash
set LEGAL_NER_MODEL_NAME=<model_name>
```

Дополнительно:

- `LEGAL_NER_MIN_SCORE` (по умолчанию `0.6`)
- `LEGAL_NER_DEVICE` (`-1` для CPU, `0` для первого GPU)

### API

- `GET /health`
- `POST /api/anonymize`
- `POST /api/document/parse` (multipart upload: `.pdf`, `.docx`)

Пример запроса:

```json
{
  "text": "ИНН 7707083893, email ivan.petrov@example.com, телефон +7 (999) 123-45-67",
  "use_ner": true
}
```

Пример загрузки документа:

```bash
curl -X POST "http://127.0.0.1:8000/api/document/parse" ^
  -H "accept: application/json" ^
  -H "Content-Type: multipart/form-data" ^
  -F "file=@contract.pdf"
```
